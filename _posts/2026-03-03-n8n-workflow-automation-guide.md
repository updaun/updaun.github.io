## n8n이란?

n8n(엔에이트엔)은 **오픈소스 기반의 워크플로우 자동화 플랫폼**입니다. 프로그래밍 지식이 없어도 다양한 애플리케이션과 서비스를 연결하여 반복적인 작업을 자동화할 수 있는 도구입니다. Zapier나 Make(구 Integromat)와 같은 자동화 플랫폼과 유사하지만, n8n은 오픈소스이므로 자신의 서버에 직접 호스팅할 수 있어 데이터 프라이버시와 커스터마이징 측면에서 더 큰 자유도를 제공합니다.

간단히 말해서, **"if this, then that"의 개념을 이용해 여러 앱과 서비스를 자동으로 연결하는 자동화 봇**이라고 생각하면 됩니다. 예를 들어 "Slack에서 특정 키워드가 포함된 메시지가 올 때, 자동으로 이를 정리해서 Google Sheets에 저장한다"는 식의 워크플로우를 만들 수 있습니다. 복잡한 조건 처리, 데이터 변환, 에러 처리 등도 모두 노-코드 인터페이스에서 처리할 수 있습니다.

## n8n의 주요 특징

n8n의 가장 큰 강점은 **400개 이상의 미리 구축된 통합(Integration)**을 제공한다는 점입니다. Gmail, Slack, Google Sheets, Stripe, GitHub, Salesforce, HubSpot, Jira, Airtable, Twilio, OpenAI 등 인기 있는 서비스들과 쉽게 연결할 수 있습니다. 또한 오픈소스 프로젝트이기 때문에 커뮤니티에서 지속적으로 새로운 노드(통합)를 추가하고 있으며, 필요하면 직접 커스텀 노드를 만들어 확장할 수 있습니다. 무엇보다 **셀프호스팅이 가능**하다는 것이 중요한 특징인데, 이는 민감한 데이터를 자신의 서버에서 안전하게 처리할 수 있다는 의미입니다.

### 주요 특징 정리

- **노-코드 인터페이스**: 드래그 앤 드롭 방식으로 워크플로우 구성
- **400+ 통합**: 거의 모든 주요 웹 서비스와 연결 가능
- **오픈소스**: GitHub에서 무료로 소스코드 확인 가능 및 커스터마이징 가능
- **셀프호스팅**: Docker, npm, CMS 등으로 자신의 서버에 설치 가능
- **무제한 실행**: 셀프호스팅 시 워크플로우 실행 횟수 제한 없음
- **복잡한 로직 처리**: 조건문, 반복문, 데이터 변환 등 프로그래밍적 로직 구현 가능
- **활발한 커뮤니티**: GitHub 스타 35,000+ (2024년 기준), 정기적인 업데이트

## n8n의 핵심 개념 이해하기

n8n을 제대로 활용하기 위해서는 몇 가지 핵심 개념을 이해해야 합니다.

### 주요 개념들

**Workflow(워크플로우)** - 여러 단계의 작업을 정의하는 전체 자동화 프로세스입니다. 워크플로우는 저장하고, 켜고 끌 수 있으며, 실행 이력을 추적할 수 있습니다. 예를 들어 "고객이 폼을 제출할 때, CRM에 저장하고 이메일을 보내고 Slack 알림을 하는" 일련의 과정 전체를 하나의 워크플로우로 부릅니다.

**Node(노드)** - 워크플로우를 구성하는 각 단계나 작업입니다. 이메일 전송, 데이터베이스 조회, API 호출, 데이터 변환 등 개별 작업 하나하나가 노드입니다. 각 노드는 입력(Input)을 받아서 처리한 후 출력(Output)을 내보냅니다.

**Trigger(트리거)** - 워크플로우를 시작하게 만드는 이벤트입니다. "새 이메일이 들어올 때", "웹훅 요청이 올 때", "매일 오전 9시", "특정 파일이 업로드될 때" 등이 트리거의 예입니다. 모든 워크플로우는 반드시 하나의 트리거로 시작합니다.

**Credentials(인증정보)** - 각 서비스에 접근하기 위한 API 키, 토큰, 로그인 정보 등을 안전하게 저장하는 곳입니다. 한 번 설정하면 여러 워크플로우에서 재사용할 수 있습니다.

**Data Flow(데이터 흐름)** - 한 노드에서 나온 출력이 다음 노드의 입력으로 전달되는 방식입니다. n8n은 JSON 형식으로 데이터를 전달하므로, 각 노드에서 필요한 데이터를 추출하고 변환할 수 있습니다.

## n8n 시작하기

n8n을 사용하기 위한 방법은 크게 세 가지입니다.

### 1. n8n 클라우드 (추천 - 초보자용)

가장 쉬운 방법은 공식 클라우드 서비스(https://app.n8n.cloud)를 이용하는 것입니다. 계정만 만들면 별도 설치 없이 바로 시작할 수 있습니다. 무료 플랜에서는 최대 10개의 워크플로우를 만들 수 있으며, 월간 실행 횟수 제한이 있지만 학습하기에는 충분합니다. 웹 브라우저에서 모든 작업을 할 수 있다는 것이 큰 장점입니다.

### 2. Docker를 사용한 셀프호스팅

Docker가 설치되어 있다면 한 줄로 시작할 수 있습니다:

```bash
docker run -it -p 5678:5678 n8nio/n8n
```

이 명령어를 실행하면 http://localhost:5678 에서 n8n에 접근할 수 있습니다. Docker를 사용하면 쉽게 설치하고 제거할 수 있어 테스트용으로 좋습니다.

### 3. npm을 사용한 직접 설치

Node.js와 npm이 설치된 환경에서 터미널에서 다음 명령어를 실행합니다:

```bash
npm install -g n8n
n8n
```

그러면 http://localhost:5678 에서 접근할 수 있습니다. 이 방법은 프로덕션 환경에 더 적합합니다.

### 추천 경로

초보자라면 **먼저 n8n 클라우드 무료 버전으로 시작**하세요. 인터페이스와 기본 개념에 익숙해진 후, 더 복잡한 작업이 필요하거나 데이터 보안이 중요한 경우에 Docker나 직접 설치로 셀프호스팅으로 전환하면 됩니다.

## n8n 실제 활용 사례

n8n의 활용 사례는 매우 다양합니다. 실제 프로젝트에서 많이 사용되는 사례들을 소개합니다.

### 1. 이메일 마케팅 자동화

특정 조건을 만족하는 이메일이 들어오면 자동으로 분류하고, Slack으로 알림을 보내고, Google Sheets에 기록할 수 있습니다. 더 나아가 고객의 이메일 내용을 분석하여 OpenAI API를 통해 자동 답장을 생성할 수도 있습니다.

### 2. 영업 파이프라인 자동화

Typeform이나 다른 폼 서비스로 들어온 고객 정보를 자동으로 Salesforce나 HubSpot CRM에 저장합니다. 또한 HubSpot의 리드 스코어가 특정 값 이상이 되면 자동으로 영업팀에 Slack 알림을 보내고 태스크를 생성할 수 있습니다.

### 3. 개발팀 협업 자동화

GitHub에 새 이슈가 생성되면 자동으로 Jira 티켓을 만들고, PR이 머지되면 Slack 채널로 알림을 보냅니다. 또한 코드 리뷰 완료 시 자동으로 배포 파이프라인을 시작할 수 있습니다.

### 4. 데이터 동기화 및 보고서 자동화

여러 서비스(Google Analytics, Stripe, Shopify 등)의 데이터를 정기적으로 수집하여 Google Sheets나 데이터베이스에 저장합니다. 이 데이터를 기반으로 주간 또는 월간 보고서를 자동으로 생성하고 이메일로 발송합니다.

### 5. 소셜 미디어 콘텐츠 배포

Blog RSS 피드나 API에서 새 콘텐츠를 감지하면 자동으로 Twitter, LinkedIn, Facebook 등 여러 소셜 플랫폼에 동시 포스팅합니다. 또한 좋아요나 댓글 수에 따라 팀에 알림을 보낼 수 있습니다.

### 6. AI를 활용한 콘텐츠 생성

Google Forms로 고객 요청을 받으면, ChatGPT API를 통해 자동으로 답변을 작성하고, 이를 이메일로 발송합니다. 또한 생성된 콘텐츠를 자동으로 Google Docs에 저장할 수 있습니다.

### 7. 재고 및 주문 관리

Shopify나 WooCommerce에서 새 주문이 들어오면 자동으로 Airtable 재고 시스템을 업데이트하고, 재고가 일정 수준 이하로 떨어지면 구매자에게 알림을 보냅니다.

## n8n과 다른 자동화 플랫폼 비교

Zapier, Make, IFTTT 등 여러 자동화 플랫폼들이 있습니다. n8n과의 주요 차이점을 정리해보겠습니다.

| 특징 | n8n | Zapier | Make |
|------|------|--------|------|
| 가격 | 무료(클라우드/셀프호스팅) | 월 $29+ | 월 $10+ |
| 오픈소스 | ✅ 가능 | ❌ | ❌ |
| 셀프호스팅 | ✅ 가능 | ❌ | ❌ |
| 통합 수 | 400+ | 5,000+ | 1,000+ |
| 무제한 실행 | ✅ (셀프호스팅) | ❌ (태스크 수 제한) | ⚠️ (플랜 따름) |
| 한국어 문서 | 적음 | 많음 | 많음 |
| 복잡한 로직 | ✅ 우수 | ⚠️ 중간 | ✅ 우수 |

**종합적으로**: 비용을 최소화하고 복잡한 자동화를 원한다면 **n8n**, 빠른 셋업과 풍부한 한국어 자료가 필요하다면 **Zapier**, 중간 수준의 복잡도와 가성비를 원한다면 **Make**가 적합합니다.

## n8n의 장점과 단점

### 장점

- **비용 효율성**: 자신의 서버에서 무료로 무제한 실행 가능
- **데이터 보안**: 민감한 데이터를 자신의 서버에서 처리하므로 프라이버시 보장
- **확장성**: 오픈소스이므로 필요에 따라 커스템 노드나 기능 개발 가능
- **유연성**: 복잡한 조건 처리와 고급 기능들이 비교적 쉬움
- **커뮤니티**: 활발한 GitHub 커뮤니티와 정기적인 업데이트
- **표준 개방**: REST API를 통해 모든 기능에 접근 가능

### 단점

- **학습곡선**: Zapier에 비해 인터페이스가 복잡할 수 있음
- **한국어 자료 부족**: 한국어 문서, 튜토리얼, 커뮤니티가 상대적으로 적음
- **운영 비용**: 셀프호스팅 시 서버 비용(AWS, Linode 등) 필요
- **유지보수**: 자신이 설치하고 업데이트를 관리해야 함
- **통합 수**: Zapier에 비해 필요한 통합이 없을 수 있음
- **초기 설정**: 클라우드 서비스보다 시작에 시간이 더 걸릴 수 있음

## 실전 튜토리얼: 첫 번째 워크플로우 만들기

n8n의 기본을 이해했다면 간단한 워크플로우를 만들어봅시다. "매일 오전 9시에 특정 URL에서 데이터를 가져와 Google Sheets에 저장하는" 워크플로우를 예로 들겠습니다.

### 단계별 구성

1. **Trigger 설정**: "Cron" 노드를 추가하고 "매일 오전 9시"로 설정
2. **데이터 수집**: "HTTP Request" 노드를 추가하고 원하는 URL을 입력
3. **데이터 변환**: "Set" 노드로 가져온 데이터를 원하는 형식으로 정렬
4. **저장**: "Google Sheets" 노드를 추가하여 해당 시트에 행 추가
5. **테스트**: "Execute Workflow" 버튼으로 실행해보기

이런 식으로 단순한 워크플로우부터 시작해서 점차 복잡한 조건과 많은 노드를 추가하면서 기술을 향상시킬 수 있습니다.

## 자주 묻는 질문 (FAQ)

### Q1. n8n은 정말 무료인가요?

네, n8n 클라우드 무료 플랜과 셀프호스팅은 모두 무료입니다. 다만 클라우드 무료 버전에는 월간 워크플로우 실행 횟수 제한(약 5,000회)이 있습니다. 셀프호스팅은 서버 비용만 부담하면 실행 횟수 제한이 없습니다.

### Q2. 코딩 경험이 없어도 할 수 있나요?

네, 완전히 가능합니다. n8n은 노-코드 플랫폼이므로 기본 개념만 이해하면 복잡한 자동화도 구현할 수 있습니다. 다만 고급 기능(JavaScript 함수, 정규표현식 등)을 사용하려면 프로그래밍 지식이 도움이 됩니다.

### Q3. 데이터 보안은 어떻게 되나요?

API 키나 비밀번호 같은 민감 정보는 n8n 내 암호화된 저장소에 저장됩니다. 자신의 서버에 셀프호스팅하면 모든 데이터를 완전히 제어할 수 있습니다. n8n 클라우드를 사용하는 경우 Twingate 같은 암호화 기술로 보호됩니다.

### Q4. 자신의 웹사이트나 API와 연결할 수 있나요?

네, 가능합니다. HTTP Request 노드로 어떤 REST API와도 연결할 수 있습니다. Webhook을 사용하면 자신의 애플리케이션에서 n8n 워크플로우를 트리거할 수도 있습니다.

### Q5. 에러가 발생했을 때는 어떻게 되나요?

워크플로우에 오류 처리(Error Handling) 노드를 추가하여 오류 발생 시 다음 스텝을 정의할 수 있습니다. 또한 실행 실패 복구(Retry), 에러 로깅 등을 설정할 수 있습니다.

### Q6. 교육용 자료나 커뮤니티는 어디에 있나요?

- **공식 문서**: https://docs.n8n.io
- **공식 커뮤니티 포럼**: https://community.n8n.io
- **YouTube**: n8n 공식 채널과 유저 채널들
- **GitHub**: https://github.com/n8n-io/n8n 에서 이슈, 토론, 예제 찾기 가능
